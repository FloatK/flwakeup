# WakeUp课程表 Flutter 重写版 — 开发指南 (for AI Agent)

---

## 1. 项目概述

**目标**：实现一款功能完整的课程表 App（参考"WakeUp课程表"的核心体验）。

**技术栈**：

| 类别 | 技术 |
|------|------|
| 框架 | Flutter 3.x + Dart 3 |
| 状态管理 | Riverpod (flutter_riverpod + riverpod_annotation，使用 @riverpod 代码生成) |
| 本地数据库 | Drift (SQLite) |
| 路由 | go_router |
| 数据序列化 | freezed + json_serializable |
| 网络 | dio (第二期引入) |
| WebView | webview_flutter (第二期引入) |
| 通知 | flutter_local_notifications (第三期引入) |
| 分享 | share_plus (第二期引入) |
| 本地设置 | shared_preferences (第三期引入) |

**架构**：Clean Architecture 分层（core → data → domain → presentation），按功能模块组织。MVP 阶段 CRUD 操作直接通过 repository 接口调用，跳过 use case 层，后续复杂业务逻辑再引入 `domain/usecases/`。

#### 3.2 数据模型

**你的任务**：在 `lib/data/models/course.dart` 中用 freezed 生成如下模型：

```dart
import 'package:freezed_annotation/freezed_annotation.dart';

part 'course.freezed.dart';
part 'course.g.dart';

@freezed
class Course with _$Course {
  const factory Course({
    required String id,              // uuid 由调用方在创建时生成，不使用 @Default
    required String name,
    required String teacher,
    String? location,
    @Default(0xFF2196F3) int color,
    @Default([]) List<TimeDetail> timeDetails,
  }) = _Course;

  factory Course.fromJson(Map<String, dynamic> json) => _$CourseFromJson(json);
}

@freezed
class TimeDetail with _$TimeDetail {
  const factory TimeDetail({
    required int dayOfWeek,          // 1=Mon, 7=Sun
    required int startPeriod,        // 1-12
    @Default(1) int duration,
    @Default([]) List<int> weeks,
    @Default('all') String singleOrDouble, // 'all', 'single', 'double'
  }) = _TimeDetail;

  factory TimeDetail.fromJson(Map<String, dynamic> json) => _$TimeDetailFromJson(json);
}
```

> **注意**：`Course.id` 不在 freezed 的 `@Default` 中生成 —— 因为 `@Default` 的值在编译时确定，会导致所有默认构造的 Course 共用同一个 id。调用方必须在创建 Course 时通过 `const Uuid().v4()` 显式传入。

执行 `dart run build_runner build` 生成序列化代码。

#### 3.4 数据库与仓库

**你的任务**：实现 Drift 数据库和仓库。

##### 3.4.1 数据库 (`lib/data/datasources/database.dart`)

**表设计**：

- `courses` 表：`id(text PK)`, `name(text)`, `teacher(text)`, `location(text nullable)`, `color(int)`
- `time_details` 表：`id(int PK auto)`, `courseId(text FK → courses.id ON DELETE CASCADE)`, `dayOfWeek(int)`, `startPeriod(int)`, `duration(int)`, `weeks(text, 逗号分隔)`, `singleOrDouble(text)`

> **weeks 字段说明**：`List<int>` 存储时转换为逗号分隔字符串（如 "1,2,3,5,7"），读取时还原。此方案在查询"某周有哪些课"时需要全表扫描后过滤，MVP 阶段数据量小可接受。若后续性能有问题可重构为关联表。

**学期配置**：新增 `semester_config` 表，存储学期元数据：

- `id(int PK auto)`, `name(text)`, `startDate(text, ISO 8601)`, `totalWeeks(int)`, `isActive(int, 0/1)`

**DAO 方法**：

- `Stream<List<CourseWithDetails>> watchAllCourses()`
- `Future<void> insertCourse(Course course, List<TimeDetail> timeDetails)`
- `Future<void> updateCourse(Course course, List<TimeDetail> timeDetails)`
- `Future<void> deleteCourse(String id)`
- `Future<SemesterConfig?> getActiveSemester()`
- `Future<void> setSemesterConfig(SemesterConfig config)`

##### 3.4.2 仓库接口 (`lib/domain/repositories/course_repository.dart`)

```dart
abstract class CourseRepository {
  Stream<List<Course>> watchAllCourses();
  Future<void> addCourse(Course course);
  Future<void> updateCourse(Course course);
  Future<void> deleteCourse(String id);
}
```

##### 3.4.3 仓库实现 (`lib/data/repositories/course_repository_impl.dart`)

实现接口，注入 AppDatabase，完成 Drift 实体 ↔ Course 模型的转换。

##### 3.4.4 验证

运行 `build_runner` 生成数据库代码，解决可能出现的错误。

#### 3.5 状态管理

**你的任务**：在 `lib/presentation/providers/course_provider.dart` 中，使用 `@riverpod` 注解（代码生成方式）定义以下 provider：

- `courseRepositoryProvider` — 返回 `CourseRepository` 实现类实例（手动创建）。
- `courseListProvider` — 使用 `@riverpod` 生成，notifier 负责：
  - 监听仓库的 `watchAllCourses()`，暴露 `AsyncValue<List<Course>>` 状态。
  - 提供 `addCourse`, `updateCourse`, `deleteCourse` 方法，调用仓库。
- `activeSemesterProvider` — 当前活跃学期配置。
- `currentWeekProvider` — 根据学期开学日期计算当前周次（1-based），若未配置学期则返回 1。

> **注意**：使用 `@riverpod` + `riverpod_generator` 的代码生成方式，而非已过时的 `StateNotifierProvider`。Provider 文件放在 `presentation/providers/` 下，属于 presentation 层。

假数据逻辑单独放在 `lib/data/datasources/sample_data.dart`，仅在调试时使用。

#### 3.6 周视图页面

**你的任务**：创建 `lib/presentation/pages/week_schedule_page.dart`。

##### UI 布局

- **顶部栏**：显示学期名称 + 第 X 周 (MM.dd - MM.dd)，左右箭头可切换周次。
- **主体**：使用 `Table` 或自定义布局实现 8 列 × 12 行网格（首列为节次标签 1-12，剩余 7 列为周一至周日）。
- **每个课程格子**：
  - 根据课程 `startPeriod` 和 `duration` 占对应行数。
  - 显示课程名和教室，背景色使用课程的 `color`。
  - 空格子显示浅灰色占位。
- **点击课程格子**：底部弹出课程详情（`ModalBottomSheet`），展示教师、教室、周次、节次等，并提供编辑/删除操作。
- **悬浮按钮(FAB)**：跳转到添加课程页面。

##### 数据

从 `courseListProvider` 读取课程，按当前周过滤。当前周通过 `currentWeekProvider` 获取。

## 4. 开发规范（Agent 必须遵守）

| 规范 | 说明 |
|------|------|
| 质量检查 | 每个任务完成后必须运行 `flutter analyze`，确保无错误 |
| 代码风格 | 代码需包含必要的 import，变量命名有意义。**不要写无意义的注释**——代码应该自解释，只在非显而易见的约束、边界情况或 workaround 处添加注释 |
| 性能优化 | 使用 `const` 构造函数和组件优化性能 |
| 状态管理 | 统一使用 Riverpod + `@riverpod` 代码生成，禁止混用其他状态管理方案。禁止使用已过时的 `StateNotifierProvider` |
| 异常处理 | 数据库操作需处理异常，Repository 层捕获 Drift 异常并转换为领域异常或返回 Result 类型，避免 crash |
| 屏幕适配 | 使用 `MediaQuery` 或 `LayoutBuilder` 保证不同屏幕正常显示 |
| 错误处理策略 | 数据层异常在 Repository 实现中捕获并记录日志；UI 层通过 `AsyncValue` 的 `error` 状态展示友好提示，提供重试操作 |

**执行顺序**：严格按照分期顺序，完成当前目标并通过验证后再进入下一个。如遇到依赖缺失等问题，先解决依赖。
